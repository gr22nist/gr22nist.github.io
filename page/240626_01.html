프로젝트 SQL(가제) 아이디어

개요

SQLite 입문자를 위한 한국어 DB 와 연습문제 제공 웹사이트

    현재 상태 - 불편한 점

  • - 한국에서 개발이나 데이터베이스 관련에 입문할 때 쓸 수 있는 한국어로 된 샘플 데이터베이스가 거의 없음
  • - SQL TUTORIAL(새 창 열림) 사이트에서 chinook.db과 샘플 연습문제를 제공하고 있으나 영어 데이터이고 인물명/지명/아이템명들이 생소하여 테이블이나 스키마, 데이터 값을 여러번 확인해야하는 어려움이 있음
  • - 언어적 이슈로 인해 공식 도큐먼트를 보는데도 자체 번역과정이 필요하고 브라우저 자동번역은 오역이나 직역의 문제가 있어 이중의 어려움을 겪음
  • - 실제 한국 회사에 취업하게 될 경우 다루게 될 데이터나 공공 API의 데이터는 다수의 컬럼이 한글로 작성된 경우가 많음
  • - 외국의 개발 관련 사이트 특성상, 한국인에게 맞지 않는 UX/UI와 사용성을 가지고 있음

    목표 상태 - 개선할 점

  • => 비전공자나 입문자가 쉽게 SQL 문법에 적응할 수 있도록 한국 정서에 맞는 한국어 샘플 데이터베이스와 그에 알맞는 연습 문제 제공
  • => SQL 문법 튜토리얼 한글화
  • => 한국어 사용자의 UX에 맞는 UI 개선으로 사용성 증가

    고려할 점

  • - 1개월~ 1개월 반의 개발 기간 안에 기획→ 디자인 → 프론트/백엔드 개발 → 보완 → 런칭의 과정을 거쳐야 하기 때문에 기본 기능은 위에 저술한 단 두가지로 개발 착수하기
  • - 생각보다 일정이 남을 경우 확장 기능을 쉽게 추가할 수 있도록 모듈화와 구조화에 신경 쓰기
  • - 개발 관련 교육 사이트이나, 친근하고 부드러운 UI 구축하기
  • - 런칭 후, 1년 가량 서비스 유지보수를 생각하고 사이드 프로젝트로 전환 계획을 세우기

프로젝트를 설정하려고 할 때 개인적으로 가장 우선순위에 두는 두가지가 있었다.

  1. 1. 실제로 사용할 수 있을 만한 콘텐츠
  2. 2. 사용자가 단 한명이라도 공익에 도움이 될 것
이 프로젝트는 그 두가지에 다 부합되었고 이 아이디어에 동조해주는 동료들이 생겨 진행해볼 수 있게 된 것이 무척 기쁘다.
실제 개발에 들어가면 좋은 일만 있진 않겠지만 꼭 완수해서 런칭하고 싶다.
